# Wireframes — Éclat Intérieur

## Principes mobile-first

- Largeur contenu max : 1200px, padding horizontal 24px (mobile) / 48px (desktop)
- Header sticky 72px, fond blanc avec bordure basse
- Sections alternent fond blanc et gris clair (#F5F5F5)
- Un seul bloc noir (citation) sur toute la page

## Header

```
[Logo texte]                    [Nav links]     [CTA rouge]
```

Mobile : hamburger → drawer plein écran noir, liens blancs, CTA rouge en bas.

## Hero (#accueil)

```
[LABEL rouge point]
TITRE MONTserrat XL (3 lignes max)
Sous-titre Poppins (max 620px)
[CTA rouge]  [CTA outline]
                    [Image hero — femme confiante, photo naturelle]
```

Layout desktop : texte 55% / image 45% en grille.

## Bloc noir (citation)

```
[LABEL gris]
```

## Méthode (#methode)

```

[01 Corps]  [02 Esprit]  [03 Confiance]
  numéro rouge   titre Montserrat   texte Poppins
```

Grille 3 colonnes desktop, 1 colonne mobile.

## Offres (#offres)

```
LABEL | TITRE | sous-titre
[Standard]  [Premium ★]  [VIP]
```

Premium : bordure rouge, badge « Recommandé », shadow rouge légère, scale 1.02.

## Témoignages (#temoignages)

```
3 cartes avec guillemet rouge, quote, nom, détail
```

Grille 3 cols desktop, carousel/stack mobile.

## À propos (#apropos)

```
[Photo coach]  |  LABEL + nom + rôle + bio + credentials
```

Photo : ratio 4:5, object-fit cover.

## FAQ (#faq)

```

5 questions, une ouverte par défaut
```

## Contact (#contact)

```
LABEL | TITRE | sous-titre
[Formulaire]     [Liste réassurance avec ✓ rouges]
```

Formulaire max-width 480px.

## Discipline rouge

- CTA header
- CTA hero primary
- Badge Premium
- Numéros méthode (01, 02, 03)
- Ligne accent bloc noir
- Guillemets témoignages
- Bouton submit formulaire

**Max 2 éléments rouges visibles par viewport.**
